Revisiting the meta-model

  • Meta-model = conceptual model to link data with theory.
  • So far, only manifest (measured) variables were considered.
  • However, some aspects of a meta-model may be difficult to quantify and measure directly.
  • This is because they represent an abstract, multifaceted concept (e.g., “biodiversity”).
  • Latent and composite variables can help with these situations.

Latent variables

Latent variables

  • Latent variables are unobserved
  • Their influence is captured by one or more indicator variable(s).
  • Direction of causality flows from latent variables to the observed variables (reflective indicators).
  • Graphically, latent variables are represented by oval node shape.

Latent variables

  • Indicator variables “emergent” from the latent variable.
  • Latent variables free of random or systematic measurement errors (unlike their indicators).
  • All indicators should be positively correlated to the latent variable (i.e., driver).

Latent variables

  • Latents can be both endogenous and exogenous.
  • Measurement model focuses solely on relating indicators to latent variables.
  • Structural model is one with directed paths between latent variables.

Latent variables

Latents often fitted in two steps:

  1. Confirmatory factor analysis
  • Multi-indicator latent variables: the same process generated multiple indicator variables.
  • Precursor to evaluation of any structural models in which the latent variables appear.
  1. Full model including latent variable
  • To estimate effect of latent variable on other variables.

Using latents to adjust for measurement error in the explanatory variables

  • Variables are often measured with error.
  • Regression approaches assume that explanatory variables are measured without error.
  • What difference does it make?

Regression dilution

  • Red = true slope, blue = OLS slope.
  • Measurement error biases the slope towards zero.
  • How to account for regression dilution?

Latents to adjust for noise in the explanatory variables

  • Possible to account for measurement error when multiple measurements were made.
  • Latent variable with multiple indicator variables.
  • Will lead to better slope estimate and higher R2.

Composite variables

Composite variables

  • Composites to specify the collective influence of a set of variables (e.g., land use).
  • Composites arise from the indicators (in contrast to latents).
  • Values are determined by causes (i.e., indicators), thus error variance is set to 0.
  • Arrows pointing from the indicators to the composite (formative or causal indicators).
  • Composites are shown as hexangular shapes.

Key differences between latent and composite variables

  • Flow of causation:
    • Latents: from construct to indicators, i.e., indicators are driven by an underlying, unmeasured process.
    • Indicators need to be correlated to each other, since a latent represents an underlying, data-generating process.
    • Composites: flow of causation is reversed and the indicators are independent entities.
    • There is no assumption about the relation between the indicators (can be positive/negative).

Key differences between latent and composite variables

How distinguish latent and composites:

  • Rule of thumb 1: if the indicators are redundant, they likely belong to a latent variable.
  • Rule of thumb 2: If the meaning of the construct changes after dropping one of the indicators, the construct likely is a composite.

Interactions

Interactions

  • In nature, things often are contingent on each other.
  • For instance, the effect of nutrients on plant growth, may depend on how disturbed the environment is.
  • Moderation: effects are different when combined.
  • In regression, the interaction coefficient estimates the effect of the product of the two predictors.
  • Interaction coefficient estimates the deviation from additivity.

Interactions

Interactions can be modelled in different ways in lavaan:

  1. Multiple groups
  2. Composites

Interactions (multi group)

sem(model, group = "age_class", data = dd)
  • Multigroup fitting allows coefficients to vary among groups.
  • Lavaan offers the “group” argument to specify for which groups coefficients should be estimated.
  • Importantly, groups have to be categorical (e.g., sex, age class).

Interactions (multi group)

Lavaan allows to introduce equality constraints on various aspects via the group.equal argument:

sem(model, group = "age_class", group.equal = c("regressions"), data = dd)

Adding constraints via group.equal:

group.equal=c("intercepts", "means", "regressions", "residuals")

Interactions (multi group)

Even more control by having the same name for different parameters:

model <- ' 
y ~ c("b1", "b1") * x1 + c("b2", "b2") * x3
x2 ~ c("b3", "b4") * x1
x3 ~ c("b5", "b5") * x2'

Same coefficients for all but the effect of x1 on x2.

Complex sampling structure

Complex sampling structure

  • Needed when data is nested, e.g.,
    • within sites.
    • within groups such as families, nests.
  • Nesting violates the principle of independent and identically distributed observations.
  • Necessary to account for data structure in the model.

Complex sampling structure

library("lavaan.survey")
  • The add-on package lavaan.survey allows the analysis of stratified, clustered or weighted data.
  • Lavaan objects are processed with a specific data structure:
    1. Initialize the design
    2. Post-process the lavaan object and compute the adjusted results.
    3. Corrected lavaan object as result.

Complex sampling structure

design <- svydesign(ids = ~ plot, strata = ~ field, 
nest = TRUE, data = dat)
summary(design)

fit.nested <- lavaan.survey(lavaan.fit = model, 
survey.design = design)
  • Needed to specify the study design
  • Here we have plots (ids) nested in fields (strata)
  • Next, we can refit the simple model from before with lavaan.survey using the specified study design as an argument.
